# Website Deployment Checklist

## Before Deployment

- [ ] All HTML files are in `docs/` folder
- [ ] CSS file is in `docs/css/style.css`
- [ ] JavaScript file is in `docs/js/main.js`
- [ ] Images are in `docs/images/`
- [ ] `.exe` file is ready (in `dist/` or uploaded to GitHub Releases)

## Screenshots

- [ ] Login screen screenshot
- [ ] Admin dashboard screenshot
- [ ] Timetable generation screenshot
- [ ] Teacher dashboard screenshot
- [ ] Save screenshots in `docs/images/screenshots/`

## Content Updates

- [ ] Update all "your.email@example.com" with real email
- [ ] Update LinkedIn/Twitter links or remove if not needed
- [ ] Verify all internal links work
- [ ] Check download button URLs

## GitHub Setup

- [ ] Create GitHub Release v1.0.0
- [ ] Upload .exe to release
- [ ] Copy release URL
- [ ] Update download.html with release URLs
- [ ] Push all changes to GitHub
- [ ] Enable GitHub Pages (Settings → Pages → Source: docs/)

## Testing

- [ ] Visit GitHub Pages URL
- [ ] Test all navigation links
- [ ] Test download buttons
- [ ] Check mobile view
- [ ] Test on different browsers

## Post-Deployment

- [ ] Share website link
- [ ] Add website URL to repository description
- [ ] Update README.md with website link
