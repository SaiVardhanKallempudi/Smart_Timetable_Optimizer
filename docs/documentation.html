Installation

Windows

  1. Download the executable

    Go to the Download page and click "Download for Windows"

    SmartTimetable-v1.0.0-Windows.exe
  2. Run the application

    Double-click the downloaded .exe file. No installation required!

    Windows Defender Warning: If Windows shows "Windows protected your PC", click "More info" → "Run anyway". This is normal for unsigned applications.

  3. First run

    The application will create a database file (smart_timetable.db) in the same folder.

Linux

  1. Download the AppImage
    wget https://github.com/SaiVardhanKallempudi/Smart_Timetable_Optimizer/releases/download/v1.0.0/SmartTimetable-v1.0.0-Linux.AppImage
  2. Make it executable
    chmod +x SmartTimetable-v1.0.0-Linux.AppImage
  3. Run the application
    ./SmartTimetable-v1.0.0-Linux.AppImage

macOS

  1. Download the DMG file

    From the Download page

  2. Open the DMG

    Double-click the .dmg file

  3. Drag to Applications

    Drag the Smart Timetable icon to your Applications folder

  4. Allow in Security & Privacy

    On first run, go to System Preferences → Security & Privacy → "Open Anyway"

Setting Constraints

Constraint Format

Constraints are entered as comma-separated values:

Format 1: Course, Section, Day, Period Range
CS101,A,Monday,P1-P3

Means: CS101 Section A must be scheduled on Monday during periods 1-3

Format 2: Course, Day, Period (Section = ALL)
MATH201,Tuesday,P2

Means: MATH201 (all sections) must be on Tuesday, period 2

Format 3: With Constraint Type
ENG301,A,Wednesday,P1-P2,Exact

Means: ENG301 Section A must be exactly on Wednesday P1-P2 (hard constraint)

Constraint Types

  • Hard: Must be satisfied (default)
  • Exact: Must be placed exactly in specified slots
  • Soft: Preferred but can be relaxed

Generating Timetables

Configuration Options

Periods per Day

Number of class periods in a day (typically 6-8)

Tip: Most schools use 6-7 periods per day

Lunch Period

Period number for lunch break (0 = no lunch break)

Example: If lunch is during 4th period, set to 4

Start Time

When the school day begins (e.g., 9:00 AM)

Duration (minutes)

Length of each period in minutes (typically 45-60)

Time Limit

Maximum time solver can run (seconds). Default: 20

  • Simple schedules: 10-20 seconds
  • Complex schedules: 30-60 seconds

Understanding the Generated Timetable

The timetable is displayed as a grid:

  • Rows: Days of the week (Monday-Friday)
  • Columns: Period numbers
  • Cells: Course names/codes
  • LUNCH: Reserved lunch period
  • Free: No class scheduled

Editing the Timetable

You can manually edit generated timetables:

  1. Double-click any cell
  2. Type new course name or "Free"
  3. Press Enter to save

Note: Manual edits may violate constraints. Validation warnings will appear if conflicts are detected.

Database Configuration

Default: SQLite

By default, Smart Timetable uses SQLite (no configuration needed)

  • Database file: smart_timetable.db
  • Location: Same folder as the application
  • Portable: Can be copied/moved

Optional: MySQL

For multi-user environments, you can configure MySQL:

Steps:

  1. Install MySQL server
  2. Create a database:
    CREATE DATABASE smart_timetable;
  3. Edit config.py:
    DB_CONFIG = { "engine": "mysql", "database": "smart_timetable", "host": "127.0.0.1", "user": "your_username", "password": "your_password" }
  4. Run init_db.py to create tables

Backup & Restore

SQLite Backup

Simply copy the smart_timetable.db file to a safe location

MySQL Backup

mysqldump -u username -p smart_timetable > backup.sql

Restore

mysql -u username -p smart_timetable < backup.sql

Troubleshooting

Common Issues

Application Won't Start (Windows)

Issue: "Windows protected your PC" message

Solution:

  1. Click "More info"
  2. Click "Run anyway"
  3. Alternatively: Right-click .exe → Properties → Unblock → Apply

Database Error on Startup

Issue: "Failed to connect to database"

Solution:

  • Delete smart_timetable.db
  • Restart application (will create fresh database)
  • Or run init_db.py manually

Slow Timetable Generation

Issue: Generation takes too long or times out

Solution:

  • Reduce number of constraints
  • Increase time limit (Settings)
  • Reduce number of courses
  • Simplify constraint complexity

No Timetable Generated

Issue: "No feasible solution found"

Solution:

  • Check for conflicting hard constraints
  • Increase number of periods per day
  • Remove some constraints temporarily
  • Verify course data is correct

PDF Export Fails

Issue: Error when exporting PDF

Solution:

  • Check write permissions to save location
  • Ensure filename doesn't contain special characters
  • Close any open PDF with same name
  • Verify ReportLab is installed (if running from source)

Frequently Asked Questions

Is Smart Timetable Optimizer really free?

Yes! It's completely free and open source under the MIT License. No hidden costs, no subscriptions.

Does it require internet connection?

No. The application works 100% offline. All data is stored locally on your computer.

Can I use it for my school/university?

Absolutely! It's designed for educational institutions of all sizes. No licensing required.

How many courses/teachers can it handle?

Tested with 500+ courses and 200+ teachers. Performance depends on hardware and constraint complexity.

Can multiple people use it on the same computer?

Yes! The multi-user system supports multiple accounts (Admin and Teacher roles).

Can I customize the algorithm?

Yes! The source code is available on GitHub. You can modify the solver logic in tools/solver_runner.py.

Is my data secure?

Yes. All data is stored locally on your machine. Passwords are hashed with bcrypt. No data is sent to any server.

Can I contribute to the project?

Yes! Contributions are welcome. Visit the GitHub repository to get started.

What if I find a bug?

Please report it on GitHub Issues. Include steps to reproduce and screenshots if possible.

Can I request new features?

Yes! Open a feature request on GitHub Issues or contribute directly.
